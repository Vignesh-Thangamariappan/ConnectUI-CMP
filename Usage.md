# CMPUI Usage Guide

This document describes how to strictly consume the `CMPUI` module within your native applications, making use of the configurations and delegations.

## Compose Multiplatform Setup

This package builds natively. To ensure the iOS components are available, you must generate the iOS `XCFramework` first:
```bash
./gradlew :shared:assembleCMPUIReleaseXCFramework
```
This generates the framework referenced by Swift Package Manager.

> [!TIP]
> **iOS Performance Optimization**: For optimal performance on high-refresh-rate iPhones (ProMotion), add the following to your app's `Info.plist`:
> ```xml
> <key>CADisableMinimumFrameDurationOnPhone</key>
> <true/>
> ```
> While `CMPUI` disables the strict sanity check to prevent crashes, adding this entry ensures the best user experience.

## iOS Usage (Swift Package Dependency)

To add this library to another Swift Package, include it in your `Package.swift`:

```swift
dependencies: [
    .package(url: "https://github.com/Vignesh-Thangamariappan/CMPUI-CMP.git", from: "1.0.0")
],
targets: [
    .target(
        name: "MyTarget",
        dependencies: [
            .product(name: "CMPUISwift", package: "CMPUI-CMP")
        ]
    )
]
```

## iOS Usage (SwiftUI)

We now provide a native SwiftUI wrapper that automatically embeds the `UIViewController` generated by Kotlin, making consumption across modern iOS applications seamless.

1. Add the package to your project. When adding products from the package, explicitly select **`CMPUISwift`**.
2. Import `CMPUISwift` explicitly in any SwiftUI View:

```swift
import SwiftUI
import CMPUISwift // Import the SwiftUI target

struct MyCMPUIScreen: View {
    
    var body: some View {
        // Step 1: Use the newly exposed CMPUIView
        CMPUIView(
            delegate: CMPUIActionHandler()
        )
        .edgesIgnoringSafeArea(.bottom)
    }
}

// Step 2: Implement the Delegate natively
class CMPUIActionHandler: CMPUIDelegate {
    func onAvatarTapped(avatar: AvatarItem) {
        print("Avatar tapped: \(avatar.id)")
    }
    
    func onAddTeamMemberClicked() {
        print("Add member tapped")
    }
}
```

## iOS Usage (UIKit)

If your app uses pure UIKit, use the direct Kotlin wrapper.

```swift
import CMPUI
import UIKit

class HomeViewController: UIViewController {
    override func viewDidLoad() {
        super.viewDidLoad()
        
        let delegate = CMPUIActionHandler(...)
        
        let composeVC = CMPUIViewControllerFactory.shared.createViewController(
            delegate: delegate
        )

        
        // Add to view hierarchy natively
        addChild(composeVC)
        composeVC.view.frame = view.bounds
        view.addSubview(composeVC.view)
        composeVC.didMove(toParent: self)
    }
}
```

## Android Usage (View API)

Android consumes the project directory as a standard module dependency instead of SPM.

```kotlin
// In your Activity/Fragment
val cmpuiView = CMPUIViewFactory.createView(
    context = requireContext(),
    delegate = object : CMPUIDelegate {
        override fun onAvatarTapped(avatar: AvatarItem) { }
        override fun onAddTeamMemberClicked() { }
    }
)

myFrameLayout.addView(cmpuiView)
```
