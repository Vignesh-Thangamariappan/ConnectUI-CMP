# ConnectUI Usage Guide

This document describes how to strictly consume the `ConnectUI` module within your native applications, making use of the configurations and delegations.

## Compose Multiplatform Setup

This package builds natively. To ensure the iOS components are available, you must generate the iOS `XCFramework` first:
```bash
./gradlew :shared:assembleConnectUIReleaseXCFramework
```
This generates the framework referenced by Swift Package Manager.

---

## iOS Usage (SwiftUI)

We now provide a native SwiftUI wrapper that automatically embeds the `UIViewController` generated by Kotlin, making consumption across modern iOS applications seamless.

1. Add the package to your project. When adding products from the package, explicitly select **`ConnectUISwift`**.
2. Import `ConnectUISwift` explicitly in any SwiftUI View:

```swift
import SwiftUI
import ConnectUISwift // Import the SwiftUI target

struct MyConnectScreen: View {
    
    var body: some View {
        // Step 1: Use the newly exposed ConnectView
        ConnectView(
            delegate: ConnectActionHandler()
        )
        .edgesIgnoringSafeArea(.bottom)
    }
}

// Step 2: Implement the Delegate natively
class ConnectActionHandler: ConnectDelegate {
    func onAvatarTapped(avatar: AvatarItem) {
        print("Avatar tapped: \(avatar.id)")
    }
    
    func onAddTeamMemberClicked() {
        print("Add member tapped")
    }
}
```

## iOS Usage (UIKit)

If your app uses pure UIKit, use the direct Kotlin wrapper.

```swift
import ConnectUI
import UIKit

class HomeViewController: UIViewController {
    override func viewDidLoad() {
        super.viewDidLoad()
        
        let delegate = ConnectActionHandler(...)
        
        let composeVC = ConnectViewControllerFactory.shared.createViewController(
            delegate: delegate
        )

        
        // Add to view hierarchy natively
        addChild(composeVC)
        composeVC.view.frame = view.bounds
        view.addSubview(composeVC.view)
        composeVC.didMove(toParent: self)
    }
}
```

## Android Usage (View API)

Android consumes the project directory as a standard module dependency instead of SPM.

```kotlin
// In your Activity/Fragment
val connectView = ConnectViewFactory.createView(
    context = requireContext(),
    delegate = object : ConnectDelegate {
        override fun onAvatarTapped(avatar: AvatarItem) { }
        override fun onAddTeamMemberClicked() { }
    }
)

myFrameLayout.addView(connectView)
```
